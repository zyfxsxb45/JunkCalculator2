# JC2 Git 提交规范

本文档规定了 JC2 项目的 Git 提交格式规范，旨在维护清晰的项目变更历史，降低系统重构与溯源成本，并要求在提交记录中留存核心算法及底层架构调整的设计逻辑。

## 1. 提交信息结构

每次提交必须严格遵循以下文本格式：

```
<type>(<scope>): <subject>
<空一行>
【动机】
（说明修改的背景或所解决的具体缺陷）
<空一行>
【原理】
（说明底层运行机制、数据结构或数学逻辑的具体变动）
```

## 2. ``<type>`` 修改类型

必须且仅能使用以下类型标识：

*   feat     : 新增系统特性或功能模块。
*   fix      : 缺陷修复（包括崩溃、死锁或计算结果错误）。
*   perf     : 性能优化（不改变外在业务逻辑的前提下降低时间或空间复杂度）。
*   refactor : 代码重构（不增加新功能、不修复缺陷，仅进行内部调用路径或命名空间调整）。
*   math     : 算理变动（专用于 CAS 引擎推导规则、积分边界、多项式算法等纯数学理论层面的更新）。
*   docs     : 文档编撰（如 HelpText.h 内容修改或项目 README 更新）。
*   test     : 单元测试集成或测试断言的修改。
*   chore    : 杂项事务（编译脚本 CMakeLists、依赖配置或代码格式化）。

## 3. ``<scope>`` 作用模块

界定当前提交影响的代码边界，必须与项目目录划分相对应：

*   frontend : 词法分析、语法递归下降生成与 AST 树构建。
*   compiler : 字节码生成、词法作用域及上下文指令分发。
*   vm       : 虚拟机栈帧控制、执行环及中断处理机制。
*   memory   : 动态类型系统基类、容器引用模型及垃圾回收器（GcHeap）。
*   math     : 大整数基础结构、高精度有理数、复平面及矩阵线性代数底层算理。
*   cas      : 计算机代数推导过程、微积分算法及极限评估器。
*   mod      : C++ 原生扩展模块层（图形、网络、系统通信等）。
*   lib      : JC2 语言标准库层（由 .jc2 脚本编写的公共库，如正则表达式、集合算法、网络封装等）。
*   arch     : 全局框架大类（全局目录重构、基础依赖头文件更替）。

## 4. ``<subject>`` 标题要求

1. 长度限制在 50 个字符以内。
2. 强制使用英文编写。
3. 强制以动词原形开头，首字母小写（例如 `fix`、`implement`、`simplify`）。
4. 结尾不添加句号。

示例：`fix(memory): resolve hash collisions in integer objects`

## 5. ``<正文>`` 详细说明

涉及核心架构、底层数据结构更替或系统数学定理定义更改的提交，正文必须具备以下要素：

*   【动机】：说明修改前系统存在的痛点、缺陷触发条件或内存/类型边界错误。
*   【原理】：阐述解决该问题所引入的结构变化、状态机切换逻辑或数学推导公式。

## 6. 标准提交示例

### 示例 1：底层类型哈希算法优化
```
perf(memory): implement shape-aware semantic hashing for matrix

【动机】
此前将 1.0 与 BigInt(1) 置入键值映射容器时，因底层浮点数与整数的内存位模式差异导致哈希分离。同时，尺寸不同的等值矩阵会发生哈希碰撞。

【原理】
1. 建立维度坍缩机制，将浮点与分数映射至复数标量域，通过容差函数吸附浮点残差与负零位模式。
2. 将矩阵边界（行数与列数）融入哈希函数计算序列。
3. 维持 Value::equals 作为哈希表查找的严格判等防线。
```

### 示例 2：标准库逻辑重构
```
feat(lib): add non-greedy matching support to regex module

【动机】
原标准库的正则表达式解析仅支持贪婪匹配，导致特定文本提取时跨越预期边界。

【原理】
修改 regex.jc2 的自动机生成逻辑，引入量词后缀 '?' 的识别规则，并将对应的 NFA 分支判定优先级后置，从而实现非强制匹配。
```

## 7. 提交模板配置

为保证提交格式统一，建议在仓库根目录创建名为 `.gitmessage` 的文本文件，内容如下：

```
<type>(<scope>): <subject>

【动机】


【原理】
```

完成后，在项目终端执行以下命令将该文件设定为仓库默认提交模板：

```bash
git config commit.template ./.gitmessage
```
